# Contributing to Wayfinder

Thank you for your interest in contributing to Wayfinder! This document provides guidelines and information to help you contribute effectively.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/wayfinder.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit your changes: `git commit -am 'Add some feature'`
6. Push to the branch: `git push origin feature/your-feature-name`
7. Create a new Pull Request

## Development Setup

See [docs/developing.md](docs/developing.md) for detailed development setup instructions.

## Code Standards

- Follow Rust coding conventions
- Use `rustfmt` for code formatting
- Run `clippy` for linting
- Write tests for new functionality
- Document public APIs

## Documentation

- Update documentation when changing functionality
- User documentation lives in `docs-site/`
- Technical documentation lives in `docs/`

## Reporting Issues

- Use the GitHub issue tracker
- Include version information
- Provide steps to reproduce
- Include relevant error messages

## Pull Request Process

1. Ensure any install or build dependencies are removed
2. Update the README.md with details of changes to the interface
3. Increase the version numbers in any examples files and the README.md to the new version
4. Your pull request will be reviewed by maintainers

## Code of Conduct

This project adheres to the Rust Code of Conduct. By participating, you are expected to uphold this code.

## Questions?

Feel free to ask questions in the GitHub discussions or reach out to the maintainers directly.
